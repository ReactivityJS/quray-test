QuRay documentation hub

QuRay is a modular reactive framework for signed data, relay-backed replication, blob transport, and native or component-based UI bindings. This HTML site complements the Markdown docs with browser navigation, embedded live demos, and manual verification pages.

Open the HTML API reference Open the interactive feature lab Open the manual verification page

What to use for what

Markdown documents

Use the Markdown files for architecture rationale, source conventions, long-form notes, and test strategy. They remain the source of truth for repo editing.

HTML API docs

Use the HTML docs to browse public API surfaces, event metadata, and module-specific examples inside a browser without opening source files.

Interactive demos

Use the demos to see bindings, relay flows, ACL denial, tombstones, and blob progress in action. The demos are lightweight manual inspection tools, not replacements for automated tests.

Operational guidance

Recommended inline HTML element: use <span> for small inline values such as alias, unread count, app title fragments, or status words inside a sentence. It preserves line height and does not introduce block-level spacing.
<p>Welcome back, <span qu-key="~/alias"></span>. You have <span qu-key="conf/app/unread" qu-format="count"></span> unread items.</p>
