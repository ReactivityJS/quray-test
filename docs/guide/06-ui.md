# 06 — UI Components

QuRay provides two complementary UI systems:

1. **Native `qu-*` attributes** — add reactive bindings to any HTML element
2. **Custom Elements (`<qu-*>`)** — purpose-built Web Components for common patterns

Both systems automatically subscribe to `db.on()` and update the DOM when data changes — whether from a local write or incoming network sync.

---

## Setup

Enable UI features in `QuRay.init()`:

```js
const qr = await QuRay.init({
  relay:      'wss://...',
  ui:         true,        // register <qu-*> custom elements
  binding:    true,        // activate native qu-key attribute scanning
  directives: true,        // enable <qu-scope>, <qu-for> directives
})
```

---

## Native `qu-*` Attributes

Add reactive database bindings to **any standard HTML element** using `qu-*` attributes.
No JavaScript needed for common patterns.

### `qu-key` — bind an element to a key

Renders `q.data` as the element's text content:

```html
<!-- Display current user's name -->
<span qu-key="~/alias"></span>

<!-- Display space title -->
<h1 qu-key="@SPACE_ID/~meta" qu-format="data.name"></h1>

<!-- Display unread count -->
<span qu-key="conf/app/unread" qu-format="count"></span>
```

### `qu-bind` — control what is bound

```html
<!-- Set src attribute from the key's data value -->
<img qu-key="~/avatar" qu-bind="attr:src">

<!-- Bind to a property (not attribute) -->
<input type="checkbox" qu-key="conf/notifications" qu-bind="prop:checked">

<!-- Bind the page title -->
<title qu-key="conf/app/name" qu-placeholder="My App">My App</title>
```

### `qu-mode` — read/write vs read-only

```html
<!-- Two-way: typing in the input writes back to the DB -->
<input qu-key="~/alias"          qu-bind="value" qu-mode="two-way">
<input qu-key="conf/app/apiUrl"  qu-bind="value" qu-mode="two-way">

<!-- Live: saves on every keystroke (default: saves on blur/change) -->
<input qu-key="~/bio" qu-bind="value" qu-mode="two-way" qu-live>
```

### `qu-format` — format the display value

```html
<!-- Show relative time ("2 minutes ago") -->
<time qu-key="~/last-seen" qu-format="rel-time"></time>

<!-- Show a formatted date -->
<time qu-key="@SPACE_ID/created" qu-format="date"></time>

<!-- Show a subfield of the data object -->
<span qu-key="~/profile" qu-format="data.bio"></span>

<!-- Show a count (integer) -->
<badge qu-key="conf/unread" qu-format="count"></badge>
```

### `qu-placeholder`

Shown when the key has no value:

```html
<span qu-key="~/alias" qu-placeholder="Anonymous"></span>
<title qu-key="conf/app/title" qu-placeholder="QuRay App">QuRay App</title>
```

## Custom Elements (`<qu-*>`)

Fully featured Web Components for common QuRay patterns.

### `<qu-bind>` — single-record template

Renders a template for a single QuBit. Template children use `data-qu-bind` to
bind to fields of the QuBit:

```html
<qu-bind key="@ROOM_ID/message/001">
  <template>
    <article class="message">
      <strong data-qu-bind="data.author"></strong>
      <p data-qu-bind="data.text"></p>
      <time data-qu-bind="ts" data-qu-format="rel-time"></time>
    </article>
  </template>
</qu-bind>
```

**Attributes:**

| Attribute | Description |
|-----------|-------------|
| `key` | The QuBit key to bind to |
| `live` | Auto-subscribe to live updates |

---

### `<qu-list>` — reactive list

Renders one template instance per QuBit in a prefix. Automatically adds/removes
items as data changes (including from remote peers).

```html
<qu-list prefix="@ROOM_ID/chat/" order="ts">
  <template>
    <li class="message">
      <qu-avatar data-pub="from"></qu-avatar>
      <span data-qu-bind="data.text"></span>
      <time data-qu-bind="ts" data-qu-format="rel-time"></time>
    </li>
  </template>
</qu-list>
```

**Attributes:**

| Attribute | Default | Description |
|-----------|---------|-------------|
| `prefix` | required | Key prefix to query |
| `order` | `key` | Sort order: `ts`, `ts-desc`, `key`, `key-desc` |
| `limit` | — | Max number of items to show |
| `reverse` | `false` | Reverse the order |
| `live` | `true` | Subscribe to live updates |

---

### `<qu-media>` — image/video/audio

Renders blob content. Handles download state, progress, and auto-display.

```html
<!-- User avatar (by key) -->
<qu-media key="~/avatar" size="64" round></qu-media>

<!-- Another user's avatar -->
<qu-media key="~BOB_PUB/avatar" size="48" round></qu-media>

<!-- Blob by hash -->
<qu-media hash="sha256url-hash" mime="image/jpeg" size="200"></qu-media>

<!-- Video -->
<qu-media hash="sha256url-hash" mime="video/mp4" controls></qu-media>
```

**Attributes:**

| Attribute | Description |
|-----------|-------------|
| `key` | QuBit key whose `data` is a blob hash |
| `hash` | Blob hash directly |
| `mime` | MIME type hint (for display mode) |
| `size` | Width/height in px (for images) |
| `round` | Apply circular crop (avatars) |
| `controls` | Show media controls (video/audio) |

---

### `<qu-avatar>` — peer avatar

```html
<!-- Show Alice's avatar with online indicator -->
<qu-avatar pub="ALICE_PUB" size="48" show-status></qu-avatar>

<!-- Current user's avatar -->
<qu-avatar pub="SELF" size="32"></qu-avatar>
```

---

### `<qu-peer>` — peer display name + avatar

```html
<qu-peer pub="ALICE_PUB"></qu-peer>
```

Renders `{avatar} {alias}` with live updates when the peer's profile changes.

---

### `<qu-profile-card>` — full profile

```html
<!-- Read-only profile -->
<qu-profile-card pub="ALICE_PUB"></qu-profile-card>

<!-- Editable (own profile) -->
<qu-user-profile pub="SELF_PUB" editable></qu-user-profile>
```

---

### `<qu-status>` — online status indicator

```html
<!-- Dot only -->
<qu-status pub="ALICE_PUB"></qu-status>

<!-- Dot + label -->
<qu-status pub="ALICE_PUB" label></qu-status>
```

Displays a green dot when the peer is online, gray when offline.

---

### `<qu-peer-list>` — list of all peers

```html
<!-- Online peers only -->
<qu-peer-list></qu-peer-list>

<!-- Include offline peers -->
<qu-peer-list show-offline></qu-peer-list>
```

---

### `<qu-delivery>` — message delivery state

Shows delivery checkmarks (like WhatsApp) for a specific message key:

```html
<qu-delivery msg-key="@ROOM_ID/chat/MSG_KEY"></qu-delivery>
```

Progresses through: clock → single tick → double tick → blue tick.

### `<qu-tick>` — static delivery icon

```html
<qu-tick state="local"></qu-tick>      <!-- clock -->
<qu-tick state="relay_in"></qu-tick>   <!-- ✓ -->
<qu-tick state="peer_recv"></qu-tick>  <!-- ✓✓ -->
<qu-tick state="peer_read"></qu-tick>  <!-- ✓✓ (blue) -->
```

---

### `<qu-sync-state>` — connection indicator

Shows relay connection state:

```html
<qu-sync-state></qu-sync-state>
```

Displays: connecting → online → offline.

---

### `<qu-grid>` — data grid

Tabular display of a QuBit prefix with sorting, filtering, and delete:

```html
<qu-grid
  prefix="@SPACE_ID/tasks/"
  columns="data.title,data.assignee,ts,key"
  sortable
  filterable
  deletable>
</qu-grid>
```

**Attributes:**

| Attribute | Description |
|-----------|-------------|
| `prefix` | Key prefix to show |
| `columns` | Comma-separated column paths (e.g. `data.title,ts`) |
| `sortable` | Enable column sort |
| `filterable` | Show text filter input |
| `deletable` | Show delete buttons |

---

### `<qu-blob-drop>` — file upload dropzone

```html
<qu-blob-drop>
  Drop files here or click to upload
</qu-blob-drop>
```

Handles file drag-and-drop, computes hash, stores to `blobs/`, and writes blob metadata QuBit.

## Directives

Directives are higher-level structural elements (enable with `directives: true`).

### `<qu-scope>` — context scope

Makes the QuBit at `key` available to all descendant elements as the current context:

```html
<qu-scope key="@ROOM_ID/posts/post-001">
  <!-- children bind to the QuBit's fields -->
  <h2 data-qu-bind="data.title"></h2>
  <p data-qu-bind="data.content"></p>
  <time data-qu-bind="ts" data-qu-format="date"></time>
</qu-scope>
```

### `<qu-for>` — iteration directive

Like `<qu-list>` but as a directive (keeps template source in DOM):

```html
<qu-for prefix="@BOARD_ID/tasks/">
  <template>
    <div class="task-card">
      <input type="checkbox" data-qu-bind="prop:checked" data-qu-key="data.done">
      <span data-qu-bind="data.title"></span>
    </div>
  </template>
</qu-for>
```

## UI Examples

### Example 1 — Simple: Show user info (HTML only)

```html
<!-- Display current user profile — zero JavaScript needed -->
<div class="profile">
  <qu-media key="~/avatar" size="64" round></qu-media>
  <h2><span qu-key="~/alias" qu-placeholder="Anonymous"></span></h2>
  <p qu-key="~/bio"></p>
  <qu-status pub="SELF_PUB" label></qu-status>
</div>

<!-- Edit profile inline -->
<div class="edit-profile">
  <label>Name: <input qu-key="~/alias" qu-bind="value" qu-mode="two-way"></label>
  <label>Bio:  <textarea qu-key="~/bio" qu-bind="value" qu-mode="two-way"></textarea></label>
</div>
```

### Example 2 — Reactive chat UI

```html
<!-- Message list: auto-updates on new messages -->
<div class="chat">
  <qu-list prefix="@ROOM_ID/chat/" order="ts" class="messages">
    <template>
      <div class="message">
        <qu-avatar data-pub="from" size="32"></qu-avatar>
        <div class="bubble">
          <strong data-qu-bind="data.alias"></strong>
          <p data-qu-bind="data.text"></p>
        </div>
        <qu-delivery data-msg-key="key"></qu-delivery>
      </div>
    </template>
  </qu-list>

  <!-- Compose -->
  <form id="compose">
    <input type="text" id="msg-input" placeholder="Type a message...">
    <button type="submit">Send</button>
  </form>
</div>

<script>
  const roomId = 'ROOM_ID'

  // Subscribe to live updates
  await qr._.sync.observe(`@${roomId}/chat/**`, () => {})

  // Send message
  document.getElementById('compose').addEventListener('submit', async (e) => {
    e.preventDefault()
    const text = document.getElementById('msg-input').value.trim()
    if (!text) return

    await qr.db.put(KEY.space(roomId).entry('chat', KEY.id()), {
      text,
      alias: qr.me.alias,
      from:  qr.me.pub,
    })

    document.getElementById('msg-input').value = ''
  })
</script>
```

### Example 3 — Complex: Reactive task board with filters

```html
<div class="board">
  <input id="filter" placeholder="Search tasks...">

  <qu-list prefix="@BOARD_ID/tasks/" order="ts" id="task-list">
    <template>
      <div class="task" data-done="false">
        <input type="checkbox" class="done-toggle">
        <span data-qu-bind="data.title"></span>
        <qu-peer data-pub="data.assignee" class="assignee"></qu-peer>
        <time data-qu-bind="ts" data-qu-format="rel-time"></time>
        <button class="delete-btn">✕</button>
      </div>
    </template>
  </qu-list>
</div>

<script>
  const boardId = 'BOARD_ID'

  // Load + subscribe
  const { off, rows } = await qr._.sync.pull(`@${boardId}/tasks/`)

  // Wire up interactivity (delegate to list)
  document.getElementById('task-list').addEventListener('click', async (e) => {
    const task = e.target.closest('.task')
    if (!task) return
    const key = task.dataset.key   // <qu-list> sets data-key on each item

    if (e.target.classList.contains('done-toggle')) {
      const q = await qr.db.get(key)
      await qr.db.put(key, { ...q.data, done: !q.data.done })
    }

    if (e.target.classList.contains('delete-btn')) {
      await qr.db.del(key)
    }
  })

  // Client-side filter
  document.getElementById('filter').addEventListener('input', (e) => {
    const term = e.target.value.toLowerCase()
    document.querySelectorAll('.task').forEach(el => {
      const title = el.querySelector('[data-qu-bind="data.title"]').textContent
      el.hidden = !title.toLowerCase().includes(term)
    })
  })
</script>
```

## Cleanup

Always unsubscribe reactive listeners when a component or view is destroyed:

```js
// ❌ Wrong — listener leaks
qr.db.on('~/alias', updateUI)

// ✓ Correct — save and call the off function
const off = qr.db.on('~/alias', updateUI)
// ... when done (page navigation, component unmount, etc.):
off()

// ✓ For custom elements — use disconnectedCallback
class MyElement extends HTMLElement {
  connectedCallback() {
    this._off = qr.db.on('~/alias', (q) => {
      this.querySelector('.name').textContent = q?.data ?? ''
    })
  }
  disconnectedCallback() {
    this._off?.()
  }
}
```
