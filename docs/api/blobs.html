Blobs and delivery

Blobs are stored under the blobs/ mount and referenced by signed blob.meta QuBits. Delivery states track where a record or blob is in the local-to-relay-to-peer path.

Blob API

await db.blobs.put(hash, arrayBuffer, { mime: "text/plain", name: "example.txt" })
db.blobs.on(hash, (entry, meta) => console.log(entry.status, meta.scope))
await db.blobs.load(hash)

Delivery tracker

db.delivery.on(messageKey, (entry) => console.log(entry?.state))
await db.delivery.set(messageKey, "queued")

Progress visualisation

The blob demo and feature lab expose upload and download progress as percentages. They combine queue progress events with blob status listeners so you can see the transition from local save to relay replica storage to peer-two download.
