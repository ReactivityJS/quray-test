UI binding

QuRay supports two complementary UI layers: native qu-* attributes on ordinary HTML elements, and component-based bindings with qu-bind and qu-list.

Preferred single-value binding

Use <span> for inline text, number, or status words inside a sentence. It keeps the line height stable and does not introduce block spacing.
<p>Welcome, <span qu-key="~/alias"></span>. You have <span qu-key="conf/app/unread" qu-format="count"></span> unread items.</p>
<input qu-key="conf/app/title" qu-bind="value" qu-mode="two-way">
<title qu-key="conf/app/title"></title>

Native binding attributes

AttributeDescription
qu-keyExact storage key or shorthand reference such as ~/alias.
qu-getValue path to read from the current record, such as data.title or ts.
qu-bindTarget binding, for example text, value, attr:title, or prop:checked.
qu-modeone-way or two-way.
qu-liveWhen present on two-way inputs, writes on every input event instead of only on change.
qu-formatOptional formatter such as count, bool, date, or json.

Template-based components

<qu-bind key="@room/messages/001">
  <template>
    <article>
      <strong data-qu-bind="data.text"></strong>
      <time data-qu-bind="ts" data-qu-format="time"></time>
    </article>
  </template>
</qu-bind>

Cleanup model

Native binding elements are cleaned up automatically when they leave the DOM. QuRay uses a MutationObserver and a binding registry to remove database listeners and DOM listeners for transient elements.
