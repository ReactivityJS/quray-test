QuDB

QuDB is the local data core. It knows about storage backends, events, tombstones, blobs, and delivery state. It does not know about remote peers directly.

Primary methods

MethodDescriptionNotes
await db.put(key, value, options?)Write a QuBit to the mounted backend and fire data listeners.Signs writes when an identity plugin is active.
await db.get(key, options?)Read one QuBit or null.Use includeDeleted: true to inspect tombstones.
await db.query(prefix, options?)Scan a prefix and return ordered QuBits.Supports order, filter, limit, and deleted visibility.
await db.del(key, options?)Create a tombstone on syncable mounts or hard-delete local-only keys.Use hard: true for local hard deletion.
const off = db.on(pattern, callback, options?)Listen for data, blob, or delivery updates.The callback receives the current record and a metadata object.

Listener metadata

const off = db.on("@room/chat/**", (currentRecord, meta) => {
  console.log(meta.event)
  console.log(meta.scope)
  console.log(meta.current)
  console.log(meta.previous)
  console.log(meta.oldValue)
}, { immediate: true })

Tombstones

await db.del("@notes/42")
await db.get("@notes/42") // -> null
await db.get("@notes/42", { includeDeleted: true }) // -> tombstone QuBit
await db.del("conf/app/temp", { hard: true })
